Main Dashboard

Overview‚Äã

The main dashboard is the central interface of the Notes App where authenticated users can view, create, edit, delete, and search their notes. It serves as the primary workspace after successful login, providing a comprehensive note management experience.

What is a Dashboard?‚Äã

A dashboard is the main user interface that:

  • Displays all user data in an organized, accessible format
  • Provides primary actions like creating, editing, and deleting content
  • Offers navigation and search functionality
  • Manages user sessions and authentication state
  • Coordinates multiple components to create a cohesive experience

File: frontend/notes-app/src/pages/Home/Home.jsx

Import Breakdown‚Äã

  • Components: Navigation, note cards, modals, and empty states

State Management‚Äã

The Home component manages multiple pieces of state to coordinate the dashboard functionality:

const [userInfo, setUserInfo] = useState(null);

State Variables Explained‚Äã

  • userInfo: Stores current user's profile information
  • allNotes: Array containing all user's notes
  • isSearch: Boolean flag indicating if search results are displayed
  • openAddEditModal: Object controlling the add/edit note modal state
  • showToastMsg: Object managing toast notification display

Authentication and User Data‚Äã

const navigate = useNavigate();

const getUserInfo = async () => {
  try {
    const response = await axiosInstance.get("/get-user");
    if (response.data && response.data.user) {
      setUserInfo(response.data.user);
    }
  } catch (error) {
    if (error.response.status === 401) {
      localStorage.clear();
      navigate("/login");
    }
  }
};

useEffect(() => {
  getUserInfo();
  getAllNotes();
}, []);

Authentication Flow‚Äã

  1. Component mounts: useEffect runs on initial render
  2. Fetch user info: Calls /get-user endpoint with stored token
  3. Handle success: Updates userInfo state with user data
  4. Handle auth failure: Clears storage and redirects to login
  5. Load notes: Fetches all user notes simultaneously

Fetching Notes‚Äã

const getAllNotes = async () => {
  try {
    const response = await axiosInstance.get("/get-all-notes");
    if (response.data && response.data.notes) {
      setAllNotes(response.data.notes);

Deleting Notes‚Äã

const deleteNote = async (data) => {
  const noteId = data._id;
  try {
    const response = await axiosInstance.delete("/delete-note/" + noteId);
    if (response.data && !response.data.error) {
      showToastMessage("Note Deleted Successfully", "delete");
      getAllNotes();
    }
  } catch (error) {
    console.log("An unexpected error occurred. Please try again.");
  }
};

Pinning/Unpinning Notes‚Äã

const updateIsPinned = async (noteData) => {
  const noteId = noteData._id;
  try {
    const response = await axiosInstance.put("/update-note-pinned/" + noteId, {
      isPinned: !noteData.isPinned,
    });
    if (response.data && response.data.note) {
      showToastMessage("Note Updated Successfully");
      getAllNotes();
    }
  } catch (error) {
    console.log(error);
  }
};

Search Functionality‚Äã

const onSearchNote = async (query) => {
  try {
    const response = await axiosInstance.get("/search-notes", {
      params: { query: query },
    });
    if (response.data && response.data.notes) {
      setIsSearch(true);
      setAllNotes(response.data.notes);
    }
  } catch (error) {
    console.log(error);
  }
};

const handleClearSearch = () => {
  setIsSearch(false);
  getAllNotes();
};

Search Features‚Äã

  • Real-time search: Searches through note titles and content
  • Search state tracking: isSearch flag manages search vs. normal view
  • Clear functionality: Returns to full note list when search is cleared
  • Query parameters: Uses URL parameters for search terms

AddEditNotes Component‚Äã

File: frontend/notes-app/src/pages/Home/AddEditNotes.jsx

A modal component that handles both creating new notes and editing existing ones.

Component Props and State‚Äã

const AddEditNotes = ({ noteData, type, getAllNotes, onClose, showToastMessage }) => {
  const [title, setTitle] = useState(noteData?.title || "");
  const [content, setContent] = useState(noteData?.content || "");
  const [tags, setTags] = useState(noteData?.tags || []);
  const [error, setError] = useState(null);

Props Explained‚Äã

  • noteData: Existing note data for edit mode (null for add mode)
  • type: String indicating "add" or "edit" operation
  • getAllNotes: Function to refresh the notes list after changes
  • onClose: Function to close the modal
  • showToastMessage: Function to display success/error notifications

Note Operations‚Äã

Adding New Notes‚Äã

const addNewNote = async () => {
  try {
    const response = await axiosInstance.post("/add-note", {
      title,
      content,
      tags,
    });
    if (response.data && response.data.note) {
      showToastMessage("Note Added Successfully");
      getAllNotes();
      onClose();
    }
  } catch (error) {
    if (error.response && error.response.data && error.response.data.message) {
      setError(error.response.data.message);
    }
  }
};

User Experience Flow‚Äã

Dashboard Loading‚Äã

  1. Component mounts: Home component initializes
  2. Authentication check: Verifies user token and fetches user info
  3. Data loading: Retrieves all user notes from backend
  4. UI rendering: Displays navbar, notes grid, and add button

Note Creation‚Äã

  1. Click add button: Opens modal in "add" mode
  2. Fill form: Enter title, content, and tags
  3. Submit: Validates and sends to backend
  4. Success feedback: Shows toast, refreshes list, closes modal

Note Editing‚Äã

  1. Click edit on note: Opens modal in "edit" mode with existing data
  2. Modify fields: Update title, content, or tags
  3. Submit: Validates and sends updated data to backend
  4. Success feedback: Shows toast, refreshes list, closes modal

Search Operations‚Äã

  1. Enter search term: Type in navbar search bar
  2. Execute search: Click search icon or press enter
  3. Display results: Shows filtered notes matching query
  4. Clear search: Click clear button to return to all notes

State Management Patterns‚Äã

Lifting State Up‚Äã

The Home component manages state for child components, following React's "lifting state up" pattern:

  • Search state: Managed in Home, passed to Navbar
  • Notes data: Centralized in Home, passed to NoteCard components
  • Modal state: Controlled by Home, affects AddEditNotes component

Callback Functions‚Äã

Parent-to-child communication through callback props:

  • getAllNotes: Passed to AddEditNotes for refreshing data
  • onSearchNote: Passed to Navbar for search functionality
  • showToastMessage: Passed to AddEditNotes for user feedback

Connection to Other Components‚Äã

Backend Integration‚Äã

  • Authentication: Validates user tokens and handles auth failures
  • CRUD operations: Create, read, update, delete notes via API
  • Search functionality: Server-side search through note content

Component Hierarchy‚Äã

  • Home (main container)
    • Navbar (search and user info)
    • NoteCard (individual note display)
    • AddEditNotes (modal for note operations)
    • Toast (user feedback notifications)
    • EmptyCard (empty state display)

State Flow‚Äã

  • Top-down data flow: State flows from Home to child components
  • Event bubbling: User actions bubble up through callback functions
  • Centralized updates: All state changes coordinated through Home component
