Principes de la Programmation Orienté Objet en Java

La Programmation Orientée Objet (POO) est un paradigme de programmation qui organise la conception de logiciels autour de données, ou objets, plutôt que de fonctions et de logique. Java est un langage purement orienté objet, ce qui signifie que presque tout en Java tourne autour des classes et des objets.

L'idée fondamentale derrière la POO est de modéliser des entités du monde réel sous forme d'objets, qui possèdent à la fois un état (données/attributs) et un comportement (méthodes/fonctions). Cette approche vise à rendre les logiciels plus modulaires, réutilisables, compréhensibles et maintenables.

Principes fondamentaux de la POO en Java​

La POO en Java est construite sur quatre piliers fondamentaux :

  1. Encapsulation
  2. Abstraction
  3. Héritage
  4. Polymorphisme

Explorons chacun d'eux :

1. Encapsulation​

Définition : L'encapsulation est le regroupement des données (attributs) et des méthodes qui opèrent sur ces données en une seule unité (une classe). Elle implique également la restriction de l'accès direct à certains composants d'un objet, ce qui signifie que la représentation interne d'un objet est cachée de l'extérieur.

Comment Java l'atteint :

  • En déclarant les variables d'instance (données) comme private.
  • En fournissant des méthodes getter et setter publiques pour accéder et modifier ces variables privées.

Exemple :

public class BankAccount {
    private String accountNumber; // Données encapsulées
    private double balance;       // Données encapsulées

    public BankAccount(String accountNumber, double initialBalance) {
        this.accountNumber = accountNumber;
        this.balance = initialBalance;
    }

    // Méthodes getter publiques
    public String getAccountNumber() {
        return accountNumber;
    }

    public double getBalance() {
        return balance;
    }

    // Méthodes setter/comportement publiques
    public void deposit(double amount) {
        if (amount > 0) {
            this.balance += amount;
            System.out.println("Déposé : " + amount);
        }
    }

    public void withdraw(double amount) {
        if (amount > 0 && this.balance >= amount) {
            this.balance -= amount;
            System.out.println("Retiré : " + amount);
        } else {
            System.out.println("Solde insuffisant ou montant invalide.");
        }
    }
}

Dans cet exemple, accountNumber et balance sont privés, ils ne peuvent donc être accédés ou modifiés que via les méthodes deposit, withdraw, getAccountNumber et getBalance, offrant un accès contrôlé.

2. Abstraction​

Définition : L'abstraction est le processus de masquage des détails d'implémentation complexes et de présentation uniquement des fonctionnalités essentielles d'un objet. Elle se concentre sur ce que fait un objet plutôt que sur comment il le fait.

Comment Java l'atteint :

  • Classes Abstraites : Classes déclarées avec le mot-clé abstract. Elles ne peuvent pas être instanciées directement et peuvent avoir des méthodes à la fois abstraites (sans implémentation) et concrètes (avec implémentation).
  • Interfaces : Des plans d'une classe. Elles ne contiennent que des méthodes abstraites (avant Java 8) et des constantes. À partir de Java 8, elles peuvent avoir des méthodes par défaut et statiques. Une classe implémente une interface, promettant de fournir des implémentations concrètes pour ses méthodes abstraites.

Exemple (utilisant une Classe Abstraite) :

// Classe Abstraite
abstract class Shape {
    abstract double getArea(); // Méthode abstraite (pas d'implémentation)

    public void display() { // Méthode concrète
        System.out.println("Ceci est une forme.");
    }
}

// Classe concrète implémentant Shape
class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    double getArea() { // Implémentation de la méthode abstraite
        return Math.PI * radius * radius;
    }
}

// Exemple (utilisant une Interface) :
interface Drivable {
    void start(); // Méthode abstraite
    void stop();  // Méthode abstraite
}

class Car implements Drivable {
    @Override
    public void start() {
        System.out.println("Voiture démarrée.");
    }

    @Override
    public void stop() {
        System.out.println("Voiture arrêtée.");
    }
}

Ici, Shape et Drivable abstraient le comportement commun getArea(), start(), stop() sans révéler les calculs ou mécanismes internes.

3. Héritage​

Définition : L'héritage est un mécanisme par lequel une classe (sous-classe/classe enfant) acquiert les propriétés et les comportements (champs et méthodes) d'une autre classe (super-classe/classe parent). Il favorise la réutilisabilité du code et établit une relation "est-un".

Comment Java l'atteint :

  • En utilisant le mot-clé extends. Une classe ne peut étendre qu'une seule autre classe (héritage simple).
  • Les interfaces peuvent également être étendues par d'autres interfaces.

Exemple :

// Classe Parent (Super-classe)
class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }

    public void eat() {
        System.out.println(name + " est en train de manger.");
    }
}

// Classe Enfant (Sous-classe) héritant de Animal
class Dog extends Animal {
    public Dog(String name) {
        super(name); // Appelle le constructeur de la classe parent
    }

    public void bark() {
        System.out.println(name + " aboie.");
    }
}

// Une autre Classe Enfant
class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }

    public void meow() {
        System.out.println(name + " miaule.");
    }
}

Ici, Dog et Cat héritent du champ name et de la méthode eat() de Animal, réutilisant les fonctionnalités communes.

4. Polymorphisme​

Définition : Le polymorphisme signifie "plusieurs formes". Il permet de traiter des objets de différentes classes comme des objets d'un type commun. Il permet à une interface unique de représenter différentes formes sous-jacentes (types de données ou classes).

Comment Java l'atteint :

  • Surcharge de méthode (Polymorphisme à la compilation) : Définition de plusieurs méthodes dans la même classe avec le même nom mais des paramètres différents (nombre, type ou ordre des arguments).
  • Redéfinition de méthode (Polymorphisme à l'exécution) : Définition d'une méthode dans une sous-classe qui a la même signature (nom, type de retour et paramètres) qu'une méthode de sa super-classe. Cela permet à une sous-classe de fournir une implémentation spécifique d'une méthode déjà définie dans sa super-classe.

Exemple (Surcharge de Méthode) :

class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) { // Méthode surchargée
        return a + b;
    }

    public int add(int a, int b, int c) { // Méthode surchargée
        return a + b + c;
    }
}

La méthode add prend différentes formes selon les arguments.

Exemple (Redéfinition de Méthode & Polymorphisme à l'exécution) :

class Vehicle {
    public void speedUp() {
        System.out.println("Le véhicule accélère.");
    }
}

class Car extends Vehicle {
    @Override
    public void speedUp() { // Méthode redéfinie
        System.out.println("La voiture accélère.");
    }
}

class Bicycle extends Vehicle {
    @Override
    public void speedUp() { // Méthode redéfinie
        System.out.println("Le vélo pédale plus vite.");
    }
}

public class PolyDemo {
    public static void main(String[] args) {
        Vehicle myCar = new Car();      // Référence polymorphique
        Vehicle myBicycle = new Bicycle(); // Référence polymorphique

        myCar.speedUp();       // Appelle la méthode speedUp() de Car
        myBicycle.speedUp();   // Appelle la méthode speedUp() de Bicycle

        Vehicle genericVehicle = new Vehicle();
        genericVehicle.speedUp(); // Appelle la méthode speedUp() de Vehicle
    }
}

Sortie :

La voiture accélère.
Le vélo pédale plus vite.
Le véhicule accélère.

Ici, myCar et myBicycle sont déclarés de type Vehicle mais se comportent différemment en fonction de leur type d'objet réel à l'exécution.

Autres concepts clés de la POO en Java​

  • Classe : Un plan ou un modèle pour créer des objets. Elle définit les attributs et comportements communs que tous les objets de cette classe auront.
  • Objet : Une instance d'une classe. C'est une entité concrète créée à partir d'une classe, représentant un élément du monde réel.
  • Constructeur : Une méthode spéciale dans une classe qui est automatiquement appelée lorsqu'un objet de cette classe est créé (instancié). Son but principal est d'initialiser l'état de l'objet.

Avantages de la POO​

  • Modularité : Découpage de systèmes complexes en objets plus petits, gérables et indépendants.
  • Réutilisabilité : L'héritage permet la réutilisation du code, réduisant le temps et l'effort de développement.
  • Maintenabilité : Plus facile de localiser, corriger et mettre à jour les problèmes grâce à la structure modulaire.
  • Évolutivité : Plus facile d'ajouter de nouvelles fonctionnalités ou d'étendre des fonctionnalités sans affecter le code existant.
  • Flexibilité : Le polymorphisme permet un code plus flexible et adaptable.
  • Sécurité des données : L'encapsulation protège les données contre la corruption accidentelle.

En résumé, la POO en Java offre un moyen structuré et puissant de concevoir et de développer des logiciels, les rendant plus robustes, maintenables et évolutifs en organisant le code autour d'objets du monde réel et de leurs interactions.
